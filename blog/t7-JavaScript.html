Ruby vs Javascript

Overall Impressions of Javascript

I though Javascript added logic and iteraction with our website. Currently we have not been introduced to this aspect of javascript yet. What we have covered so far seem to be a review of what we have learned in Ruby but with different rules and syntax. Now Javascript and Ruby as similar to many ways. They if loops, iterators, arrays, booleans, variable scopes, math and methods to name a few. While all this items operate similar to how they would in ruby, the syntax is the first thing I found different. Instead of do..end they have { }. instead of def function_name() end they have function name(){};. I could go on and on, however today I wanted to focus on Ruby classes vs javascript constructors.

I was really surprised when we ran across constructor functions early in the week. As we created them I had no idea what they were supposed to do. It was not until we printed some values to the screen that the light bulb moment came about -- "hey these similar to ruby classes..."

Ruby Classes

class Cart
    attr_reader :basket
    def initialize
        @basket = [ ]
    end
    def add(fruit)
        basket.push(fruit)
    end
end

class Item
    attr_reader :name, :qty
    def initialize(name,qty)
        @name = name
        @qty = qty
    end

apple = Item.new("apple",3)
milk = Item.new("milk", 1)
cart = Cart.new
cart.add(apple)
cart.add(milk)

p cart.basket
#< Item:0x000000016ed6a0 @name="apple",
@qty=3>, #< Item:0x000000016ed448 @name="milk",
@qty=1>]

p apple.name #=> "apple"

Javascript Constructor function

function Cart(){
    this.basket = [ ];
    this.add = function(fruit){
        this.basket.push(fruit);
    };
};
function Item(name, qty){
    this.name = name;
    this.qty = qty;
};

var apple = new Item("apple",3);
var milk = new Item("milk",1);
var cart = new Cart()
cart.add(apple);
cart.add(milk);

console.log(cart.basket)
//-> [ { name: 'apple', qty: 3 }, { name: 'milk', qty: 1 } ]

console.log(apple.name) //-> apple








Discussion

The above code does exactly the same thing, one is done is javascript the other in ruby. Everyting is in the same order so you can match code side by side.

Here is a list breakdown of the major differences

  • this.name is acting simliar to the @name.
  • There is no class keyword in javascript, you, the this.X within a function hints that this function is constructor function. Also the name is capitalized.
  • a function inside a function that is assigned to a variable is called a method function in ruby declare a method inside a class is just a method.
  • to access variables outside a ruby class you must assigned them to attr_Reader :name, :qty. for Javascript its alittle more lineant and you can access them with just . notation
  • printing items to the console in ruby is down with puts, print or p in javascript the only way I know how to do it is w/ console.log()
