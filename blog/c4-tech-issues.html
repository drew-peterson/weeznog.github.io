What is Web Acessibility?

Web Accessibility and the people it effects

When your considering how to best design a website it is important to think about designing with web accessibility in mind. When I think about the individual that web accessibility affects I immediately consider people with visual or motor impairments. People with visual and motor impairments often have programs called screen readers to help interpret websites. However web accessible content should also focus on being easy to understand, read and use. When you start designing your website, making sure it's user friendly will make the website easier to use for both the visually and tech impaired individuals.

What are screen readers?

Screens readers are programs that interpret the website and output speech when combined with specific keystrokes or mouse movements. There different ways screen readers can provide audio user interfaces, the two most common types are linearization and content skimmers. linearization is referring to going one item by a time with audio readouts at each item. Skimmers are more advanced forms of navigation that can be programed to look for specific website tags such as links, buttons, headers, navigation, paragraphs etc. The user would specify how they want to navigate the page and then continue to skip from link to link or header to header so they can navigate quickly to the information they want.

What are the stumbling points for screen readers?

Because users of screen readers rely on audio feedback its important that your website has meaningful text. Here are several examples of stumbling points for screen readers.

  1. When a screen reader comes across an image, its not going to describe the image but look for alternative text to read out to the user. When the alternative text is not available the screen reader has no idea what that image is about.
  2. When creating a new account you are often asked for your name, email, and password before you can create your new account. When you see the label "email", you understand the website wants your email address. Screen readers look for specific tags, they do not consider what appears close to the box they want specific label,alt or id tags so they know what the forum box is point to. If tags are not present, the screen reader will not give the proper feedback to the user.
  3. Finally just imagine that html headers are used as paragraphs or do not exist at all. If a user is scimming a website using header tags, and none exist or they are not used properly the user will lose all sense of website heirachy.

Quick solutions to ensure your website is web accessible

If you dont cut corners from the beginning your website should be web accessible however there are several items to consider when making a website:

  1. Make sure text is large enough and has high contrast against the background
  2. Use proper HTML5 element syntax and make sure to utilze different levels of h1-h6 header types
  3. Properly format tables and forums with there corresponding headers,lables and id's
  4. All links should include alternative meaninful text
  5. If you are using visual plugin or content, make sure there is a alternative text version to represent the information
