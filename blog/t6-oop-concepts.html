Modules VS Classes

Introduction

Classes and Modules are similar and different in many ways. They both store methods inside them. Those methods can be called using . operation, and they both can be used in different files using keywords like require.

However the biggest different between a Class and Module is that classes have instance and class variables/methods. Instance methods and variables are particully usefully because they can be used over and over again with new instances of the class without changing anything inbetween the different instances. modules cannot be intialized, and their variables and methods will have affect will be overwritten and changed if used in mutliple areas. Lets look at some examples...

Classes

class Cart
    attr_reader :basket
    def initialize
        @basket = [ ]
    end
    def add(fruit)
        basket.push(fruit)
    end
end

Modules

module Count
    def how_many(amount)
        puts "you have #{amount.length} items in your basket"
    end
end




Mixing

Lets create a new method inside the Cart class that tells use if we have enough items in our basket. The module Count holds a method that will tell use how many items we have, lets include that in our method using the following code. class Cart
Methods from above...
    include Count
    def have_enough
        puts "you have 5 items in your basket: #{how_many(basket) == 5}"
    end
end

Calling the methods and Modules

using the keyword include Count will allow the have_enough method to use the how_many method to count have many items are in the basket. Classes have their own form of include called require "file-location.rb" which will take an entire seperate file of classes and methods and include them in the file with the require. Now lets print some things to the screen and see whats happens. cart = Cart.new
cart.add("apple")

calling cart.have_enough will call the module and print out 2 statements since the module runs first then how_many(basket) then == 5 will print out its statement.
