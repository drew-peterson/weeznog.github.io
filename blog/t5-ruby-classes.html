Lamp Class

class Lamp
    def initialize(watts, shade_style)
        @watts = watts
        @shade_style = shade_style
        @light_span = rand(1..5)
        @wear_and_tear = 0
        @powered = false
    end

    def on
        if @powered == true
            puts "Its already On"
        else
            puts "You Turn the Lights On"
        @powered = true
        end
    end

    def off
        if @powered == false
            puts "Its already off"
        else
            puts "you turn the Lights off"
            @powered = false
            broken_light
        end
    end

    def broken_light
        @wear_and_tear += 1
        if @light_span == @wear_and_tear
            puts "your light broke.. Its time to change the bulb..."
            puts "you take off your #{@shade_style} and
             replace it with another #{@watts} watt bulb."
        end
    end
end

Talkin about it

Setting up our Lamp Class

  • Instance Variable: commonly seen w/ a @var_name notation, an instance variable exists within a single instance of the class and is shared by all instance methods. This differs from a local variable that only exists at the scope it is declared unless passed along as an argument.
  • initialize method: special method that can be used inside a Class. When you create a new instance of a class with lamp = Lamp.new(args..) the instance method will run automatically, so it a good idea to set up all your variables and collections to be used by the class.

    • Check out the initialize method to the left - Notice the syntax of how we created the intialized method. We define it with initialize and have pass 2 arguments to it, Watts and shade_style. If you scroll down to the bottom of the class you will see a line that says lamp = Lamp.new(50,"square" the 50 is pointing to the watts and the "square" is pointing to shade_style. Since we are creating a lamp class, what do most lamps have that make them lamps?

  • lighbulb with a watts : @watts
  • a lamp shade which takes on some form or another : @shade_style
    • So 50 and square become too instance variables that make up our lamp class. We see the aruments being passed in from creating the lamp class converted into instance variables with these lines of code @watts = watts, @shade_style = shade_style

    Inside our intialize method your will also notice we have 3 additional instance varirables being created.

  • @light_span = rand(1..5)
    • Every light bulb has a different amout of uses until it stops working. This variable creates a random number between 1 and 5. We dont know when the light will break, but we do know it will break someday. @light_span holds the value of how many times the light can be turned on and off until it breaks.
  • @wear_and_tear = 0
    • This variable acts as a counter. Everytime you turn off the lamp, wear_and_tear goes up by 1. Just like in real life, everytime you use something you are adding wear and tear on that object. When @wear_and_tear == @light_span the lamp breaks and we have to replace it.
  • @powered = false
    • Can you turn a light on if its already on? NOPE... In our instance method we will use this variable to check to see if the lamp is already ON or OFF. Powered is referring to YES its ON when it equals true

Using Instance Methods

  • def on : This instance method will run when we want to turn the lamp on. You notice we have a check here : if @powered == true; puts "its already on". We talked about @powered before, this check will ensure that if the lamp is already on @powered = true then we cannot turn the lamp on. If the Lamp is not on, the check will fail and everthing under else will run. Because we are turning the lamp from off to on @powered now = true.
  • def off : This instance method runs identially to def on except its talking about turning the lamp OFF. We have a check, which checks to see if @powered == false, or is the lamp currently turned off. If its ON, @powered == flase, light is turned off. However will will also notice under this line we have broken_light. If you scroll down alittle bit more you will see we have a method called def broken_light. Calling a method inside another method means that when def off is called, at the very end it will automatically class the method def broken_light.
  • def broken_light : Broken light a method I am using to check when the lamp will finally break. The first line @wear_and_tear += 1 adds a 1 to the variable. Just like in the real word, when you use something you add wear_and_tear on that object. The next line if @light_span == @wear_and_tear is another boolean check. Again we never know when our light is going to break. @light_span was set to a random number in our intialize method. When the wear and tear finally catches up to the light span the lamp will break. When the lamp breaks we get alittle message giving some details.
    An Important thing to note is that we can use instance variables inside another method. We created @watts in the intialize method, but we are able to use that variable here inside broken_light. Like the definition of instance variables says, instance variables are shared among all instance methods.

At the very bottom of the written class we see lamp.on. So the line before it creates a new instance of the lamp class and calls in lamp. lamp.on is calling the def on method or turning the lamp on. We then use lamp.off to turn the lamp off. when we try to turn the lamp on when its already on, we get our little message "its already on". If you run this in your code a few times you will eventally find a lamp_span that breaks only after one cycle. If the @light_span = 5, you would have to add lamp.on... lamp.off 5 times before it breaks.
